# 049 - Packet Parser Fuzz And Property Tests

## Description
Add fuzz or property-style tests for WireGuard packet framing, shared-port relay demux metadata, PNet internal marker parsing, and PNet padding rejection.

The goal is to harden parser boundaries against malformed, truncated, oversized, ambiguous, and randomly generated byte sequences.

## Playbook
- `WireGuard framing`: fuzz initiation, response, cookie reply, and transport packet headers and sizes.
- `MAC1 demux`: property-test zero-match, single-match, and ambiguous-match routing decisions without decrypting payloads.
- `Receiver index`: property-test learned fast-path keys and expiry behavior for `remote_endpoint + receiver_index`.
- `PNet marker`: fuzz first-byte marker classification for IPv4, IPv6, PNet, and invalid/reserved payloads.
- `Padding`: property-test padding count `0..15`, empty payload representation, truncation, and non-zero padding rejection.
- `Corpus`: keep deterministic seed cases for every accepted and rejected boundary condition.

## Research

Parser and boundary surfaces are not ready yet: this issue depends on framing, raw helper, padding, and shared-port demux APIs. The recommended implementation path is deterministic xUnit corpus coverage first, with fuzz/property-style tooling only if the existing byte corpus proves insufficient.

## Related Tracking
- Related: #037 WireGuard packet framing and TAI64N replay tracking.
- Related: #043 mesh channel raw payload helpers.
- Related: #044 PNet frame padding and default protobuf header.
- Related: #046 shared-port relay registration and demux.

## Scope
- In scope: fuzz/property test harness, deterministic seed corpus, parser crash rejection, accepted/rejected boundary assertions.
- Out of scope: implementing parser behavior itself, changing protocol formats, long-running soak tests, external internet-facing fuzz infrastructure.

## Acceptance Criteria
- Tests exercise malformed and random byte sequences without parser crashes.
- WireGuard packet size/type parsing rejects truncated, oversized, and reserved packets.
- MAC1 demux properties cover zero, one, and multiple candidate matches.
- Receiver-index fast-path properties cover endpoint scoping and expiry.
- PNet marker and padding properties cover all low-nibble padding values and invalid markers.
- Seed corpus includes representative accepted and rejected packets for regression coverage.
